Concept

In summer, I was on vacation with my friends. On the evenings, we played a simple mobile game which only required one mobile phone with the app installed.

A single question is displayed on the screen, for example: “Who will get married first?”. The person who is currently holding the phone gives it to the person which is most likely to be the answer to the question in their opinion. They do so without the other people seeing the question.

If the person who got the phone agrees, they read out the question (and maybe chat about it in the group). If not, they pass it to the person they think is more fitting.

The game was called 100 Questions.

I liked the game because it was so simple and encouraged discussions about interesting topics. Sadly, it only contained 100 questions, so we soon had finished all of them. Also it displayed a lot of ads and only contained questions about some specific topics, not all participants felt comfortable talking about.

That inspired me to write a similar game by myself using our own custom set of questions. This way, the game is more tailored to the group and we can extend the question pool whenever we like.

Recently, I took my first steps into creating games using the Godot Engine, so this was an awesome opportunity to create my first simple game with it.

Process

I started by breaking down the game into its most basic components. In the official Godot Engine Documentation, they describe how they imagine you to design a game in Godot, so I tried to follow this explanation.

In the end, the game was so simple, that it didn’t really benefit from creating such a diagram. The main game only consists of some text on a randomly colored background. So I ended up creating 1 scene for the game and 2 menu scenes.

For the main game (text on colored background) I defined a set of colors to pick from and created a function returning a random color different from the current one. The questions are stored in a file, because I wanted to make it easily extendable. An additional benefit of storing them in a file is that I can create a Google Doc or similar and let my friends add questions without opening the engine. The file would also serve as a save file to persist between play sessions. One of the menus allows for adding questions to the file, those are also stored eternally. To ensure that the data persists between play sessions, I created a file in the user:// directory, which is stored locally on the user’s device.

I had recently learned about the power of UI Containers in Godot, so I created all the menu items out of them. This ensures that the menu looks good on any device.
