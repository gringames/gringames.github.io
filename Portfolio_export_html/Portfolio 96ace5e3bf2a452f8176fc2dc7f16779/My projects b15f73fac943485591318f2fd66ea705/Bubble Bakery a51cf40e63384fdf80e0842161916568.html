Concept

Bubble Bakery is a visual novel made for the FemDevs game jam. The themes were ‚ÄúUnderwater‚Äù and ‚ÄúChristmas‚Äù, so the game is set in an underwater bakery the night before Christmas. The player experiences a cute and short story through the eyes of the bakery shopkeeper by clicking through dialogues with the customers.

Bubble Bakery is in debt and the short time frame to pay them of is over, so your bakery is about to be bought by the big fast food chain Fishy Business.

The team was comprised of an artist, a composer, a game designer, a narrative designer and me, the programmer.

Contributions

I was the only programmer in this project, so I did any coding tasks. These include:

Before writing any code, I defined the possible actions I need to cover in the timeline. These were: talking, moving into and out of the bakery and ordering. As I said in the concept section, there are no orders in the final submission, but technically they are a possible action. Now that I had laid out all the actions, I planned on how to encode them in the text file. Each action would end up in a separate line to make things easier in code. The script would get a reference to the timeline of the current scene and read this very file line by line. A line could either start with a comment, indicated with a # at the start, or with one of the possible actions. Comment lines as well as empty lines are simply skipped by the parser. This makes it easier to structure the file.

An action line always starts with the action type, being TALK, ENTER, EXIT or ORDER, respectively. I also added a STOP command that immediately stops the timeline execution when found in any line. This made it a lot easier to test parts of the timeline without having to comment out or remove the following lines.

After the action type, the name of the character doing that action follows. They are separated by ¬ß as this will never occur in any dialogue or name. I intuitively split the line at any whitespace " " initially, but I quickly ran into problems there, because I needed to have the text content of the dialogues in the TALK action in one single string but the parser would split up all words because of the spaces.

Now we have extracted the action type and the name out of the line. If there is anything left in the line, it is stored in an extra array called arguments.

Now to the action processing: I created an interface called Action which had an abstract method Handle(string[] arguments). I then created a class for each Action implementing that interface and the Handle() method. Depending on the extracted action type, the parser delegates character name and arguments to the respective action object.

Depending on the action, the timeline might have to wait for the action to finish before processing the next line. In the game jam version, I did this by providing a public method HandleNextLine() in the TimelineParser.cs which can be called by the action objects as soon as they are done with their action. In some cases, I additionally used events the actions subscribed to. That was needed, because the Movement was implemented using coroutines and the dialogue had to wait for user input.

    using System;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.SceneManagement;

namespace Timeline
{
    public class TimelineParser : MonoBehaviour
    {
        [Header("Actions")] [SerializeField] private MoveAction moveAction;
        [SerializeField] private OrderAction orderAction;
        [SerializeField] private TalkAction talkAction;

        [Header("File")] [SerializeField] private TextAsset timelineFile;
        [SerializeField] private string commentCharacter = "#";
        [SerializeField] private string splitCharacter = "¬ß";
        private string[] _lines;

        // ACTIONS
        private const string ENTER = "ENTER";
        private const string TALK = "TALK";
        private const string EXIT = "EXIT";
        private const string ORDER = "ORDER";
        private const string STOP = "STOP";

        // PARSING
        private int _lineIndex = 0;
        private int _lineCount;


        private void Awake()
        {
            InitializeTimeline();

            if (_lines[0] == STOP) return;

            ParseNextLine();
        }

        private void InitializeTimeline()
        {
            _lines = SplitTextAtNewLine(timelineFile.text);
            _lineCount = _lines.Length;
        }

        private static string[] SplitTextAtNewLine(string text)
        {
            return text.Split(new[] {Environment.NewLine},
                StringSplitOptions.RemoveEmptyEntries);
        }

        public void ParseNextLine()
        {
            if (_lineIndex == _lineCount)
            {
                Debug.Log("end of timeline reached");
                SceneManager.LoadScene(2);
                return;
            }

            var line = _lines[_lineIndex];
            _lineIndex++;

            // skip line if it is a comment
            if (SkipLine(line))
            {
                ParseNextLine();
            }
            else
            {
                HandleLine(line);
            }
        }

        private bool SkipLine(string line)
        {
            return line.StartsWith(commentCharacter)
                   || line.StartsWith(" ")
                   || line.Length == 0;
        }

        private void HandleLine(string line)
        {
            string[] contents = line.Split(splitCharacter);

            var action = contents[0];
            if (action == STOP) return;
            
            var arguments = GetArguments(contents);
            
            
            switch (action)
            {
                case ENTER:
                    arguments = AddTypeToMoveActionArguments(arguments, true);
                    moveAction.Handle(arguments);
                    break;
                case EXIT:
                    arguments = AddTypeToMoveActionArguments(arguments, false);
                    moveAction.Handle(arguments);
                    break;
                case ORDER:
                    orderAction.Handle(arguments);
                    break;
                case TALK:
                    arguments = ConvertAllTalkLinesToArguments(arguments);
                    talkAction.Handle(arguments);
                    break;
                default:
                    Debug.LogError($"unknown action: {action}");
                    break;
            }
        }

		// ...

		}
}

Dialogue Scrolling and Display

The requirements for the dialogue were as follows:

  1. Show the textbox with the correct character name.

Showing the textbox was simply done by setting the textbox object active. I created an extra class for managing the dialogue, holding references to both the text object holding the character‚Äôs name and the one containing the actual dialogue part. The text of the name field is set to the character name passed to the TALK's Handle() method.

  1. ‚ÄúAnimate‚Äù the text: The contents appear letter by letter. This can be skipped by the player clicking the left mouse button.

In the DialogueManager.cs I created a coroutine typing out each char of the passed dialogue string at a fixed speed. If the player presses the mouse button, all coroutines are stopped and the text is set to the whole content passed to the method.

  1. When the current dialogue content is written out fully, the player needs to click to go to the next part.

For this, I again check for mouse input. If a click happens, I tell the TimelineParser.cs to continue parsing.

    using System;
using System.Collections;
using TMPro;
using UnityEngine;

namespace Dialogues
{
    public class DialogueManager : MonoBehaviour
    {
        [Header("TMP Objects")] [SerializeField]
        private GameObject dialoguePanel;

        [SerializeField] private TextMeshProUGUI dialogueName;
        [SerializeField] private TextMeshProUGUI dialogueText;


        [Header("Text Scrolling")] [SerializeField]
        private float timeBetweenLetters = .01f;

        private string _currentContent;

        public Action OnFinishedTyping;
        
        
        public void ShowPanel()
        {
            dialogueName.text = "";
            dialogueText.text = "";
            dialoguePanel.SetActive(true);
        }

        public void HidePanel()
        {
            dialoguePanel.SetActive(false);
        }

        public void SetName(string characterName)
        {
            dialogueName.text = characterName;
        }

        
        public void AnimateContent(string content)
        {
            _currentContent = content;
            dialogueText.text = string.Empty;
            StartCoroutine(nameof(Type), content);
        }

        public void SkipTyping()
        {
            StopAllCoroutines();
            dialogueText.text = _currentContent;
            
            OnFinishedTyping?.Invoke();
        }

				// inspired by: https://www.youtube.com/watch?v=8oTYabhj248
        private IEnumerator Type(string content)
        {
            foreach (var c in content.ToCharArray())
            {
                dialogueText.text += c;
                yield return new WaitForSeconds(timeBetweenLetters);
            }
            
            OnFinishedTyping?.Invoke();
        }

    }
}

Movement

There are two possible movements a character can perform: Entering the bakery and exiting it. Both is handled by the MoveAction.cs which takes a boolean determining if the character should go in or out. The TimelineParser.cs adds either true or false to the arguments[] of the action, depending on whether it read EXIT or ENTER, respectively.

In the scene, I placed 3 transforms; one in the bakery in front of the counter and 1 at each side of the screen. For ENTER, the corresponding character is positioned at the right point and moved towards the center point. For EXIT, the character is moved from the center to the left point.

For the movement, I used MoveTowards() in a coroutine, so I get a smoothly interpolated movement. As soon as the character reached a position close enough to the center point, I tell the TimelineParser.cs to continue parsing.

I need the parser to wait until the movement is finished, otherwise i.e. the dialogue could start without the character having arrived at the counter.

Here is how the movement is implemented:

  • [Script] MovingObject.cs
    using System;
using System.Collections;
using UnityEngine;

public class MovingObject : MonoBehaviour
{
    public Action OnFinishedMoving;

    [Header("Properties")] [SerializeField] private float speed = 0.1f;

    private const float Threshold = 0.1f;


    public void Enter(Transform entryPoint, Transform targetPoint)
    {
        transform.position = entryPoint.position;
        StartCoroutine(nameof(MoveTo), targetPoint);
    }

    public void Exit(Transform exitPoint)
    {
        StartCoroutine(nameof(MoveTo), exitPoint);
    }

    private IEnumerator MoveTo(Transform end)
    {
        var endPoint = end.position;

        while (!AreCloseEnough(transform.position, endPoint))
        {
            transform.position = Vector3.MoveTowards(transform.position, endPoint, speed);
            yield return new WaitForFixedUpdate();
        }
    
        OnFinishedMoving?.Invoke();
    }

    private static bool AreCloseEnough(Vector3 pos, Vector3 goalPos)
    {
        return Mathf.Abs(pos.x - goalPos.x) < Threshold;
    }
}
