Context & Description

VR Gate was developed in the context of “Virtuelle Trainingswelten”, a university course focusing on creating a gamified virtual training and working out its benefits compared to a non-digital solution.

In the past year, I’ve attended a talk about a digital training taking place in a chemical lab and how various scenarios can be simulated in VR with ease, which in turn enhances the trainee’s learning success. Being able to switch between scenarios with a single click is especially useful for practicing very rare scenarios for which one would have to wait a long time to happen in a real environment.

With this in mind, I proposed to create a digital training for flight attendants which could be used as a stand in for the practical part. Me and my team then narrowed down the levels to basic tasks like “Help the passengers to put their luggage in the correct spaces” or “grab the right items during the safety introduction”. We ended up with a small tutorial, 5 single training levels focusing on one particular task and an exam mode where trainees have to get through all levels in a row to pass the exam.

This game, of course, has no ambitions to be used in real-world flight attendee trainings. It is only a small proof of concept playing with the idea of a gamified training and how it could look like.

Process

Level Selection

First of all I had to figure out how to make a canvas appear in world space. As this was a VR project, I refrained from using simple clickable buttons but rather placed the level icons on a wall in the start room. The player can walk around freely there and point on a level using the VR controllers. This adds to the immersion in the virtual environment.
Apart from that, I did simple scene wiring as in any other level selection where each level has its own scene.

Eric the Steward

We wanted to provide an option to leave a level and return to the main menu. We could have made a normal VR pause menu, but I had this idea in my mind that I had to try. I always prefer to create something that fits the world and context of the game where it is possible. And when I think of my experiences with taking a flight, I have the picture of the stewards greeting the passengers at the plane’s door in mind. So my idea was to put a steward in front of the cockpit that the trainee could interact with to open some kind of pause menu.

We downloaded a simple steward model and placed it in front of the cockpit in every level taking place inside the plane. I can't remember why exactly we named him Eric, but that's how he's called.

There is a little dialogue window attached to Eric that should pop up as the player approaches him. To achieve this, I placed a trigger box around the steward.

He is also featured in the tutorial level standing in front of the escalator to the plane. Eric quickly introduces himself to make the player familiar with this way of traversing through the levels. I scaled up the collider so that the player has to enter it when approaching the plane. This way I ensured that the player will read this message before playing one of the levels.

“Needs” Level

The concept of this stage was that the player has to fulfill the passenger’s needs. I broke down the level in simple requirements I can work through:

  • Display a text box explaining what to do in this stage.
  • Have the plane filled with passengers.
  • Once in a while, a passenger says that they need something.
  • The trainee has to deliver the correct item.
  • Show a text box with a response to the item.
  • For exam mode: trainee has to fulfill 5 needs correctly in order to pass the level.

Each level has a small text box showing the objectives at the start, so I started with that. I needed to include both a practice and an exam version of the instructions because we decided that in practice mode the player can play infinitely to train this very task. But in exam mode there are strict requirements that have to be reached to pass the level. In this case, helping five passengers.

    using UnityEngine;

public class ColliderDialogueManager : MonoBehaviour
{
    [SerializeField] private Canvas textbox;
    [SerializeField] private Canvas exclMark;


    private void Awake()
    {
        textbox.gameObject.SetActive(false);
    }

    private void OnTriggerEnter(Collider other)
    {
        if (!other.gameObject.CompareTag("Player"))
        {
            return;
        }

        textbox.gameObject.SetActive(true);
        exclMark.gameObject.SetActive(false);
        gameObject.SetActive(false); // disable collider as well
    }
}
    using TMPro;
using UnityEngine;

public class NeedsTextbox : MonoBehaviour
{
    [SerializeField] private Canvas textbox;
    [SerializeField] private TextMeshProUGUI text;

    private readonly string _examText =
        "Hilf 5 Passagieren, indem du ihnen das gewünschte Objekt aus der Küche bringst.";

    private readonly string _practiceText =
        "Bringe den Passagieren, was sie wollen. Zum Verlassen sprich mit dem Steward hinter dir.";

    private void Start()
    {
        text.text = LevelSwitch.IsExam ? _examText : _practiceText;
        textbox.enabled = true;
    }

    // OnClick for Tutorial "OK" Button
    public void DismissTutorial()
    {
        textbox.gameObject.SetActive(false);
        textbox.enabled = false;
    }
}

During testing, I quickly realized that I had to change how I picked the passengers who ask for help. The problem was that the player could not reach any passengers that were not sitting in an aisle seat. So I did two things: First I split the passengers into static ones and those that could ask for something. Only the latter would have a need collider, a text box, and exclamation mark attached as their child objects. Those would only spawn in the aisle seats and added to the list.
